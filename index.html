1) Recon — passive → active

OSINT and basic DNS lookups to gather targets and context.

whois / dig / nslookup
whois example.com
dig +short example.com A
dig @8.8.8.8 example.com ANY
nslookup -type=mx example.com

# OSINT harvest
theharvester -d example.com -b google -l 500
shodan host 1.2.3.4

2) Subdomain & DNS enumeration

Discover subdomains and DNS records.

amass / subfinder / crt.sh
# amass (passive + active + brute)
amass enum -d example.com -o amass.txt
amass enum -d example.com -brute -o amass_brute.txt

# subfinder (fast passive)
subfinder -d example.com -o subfinder.txt

# crt.sh via curl + jq
curl -s "https://crt.sh/?q=%25.example.com&output=json" | jq -r '.[].name_value' | sort -u

# dnsrecon / dnsx
dnsrecon -d example.com
cat subdomains.txt | dnsx -a -resp -silent -o dnsx_a.txt

3) Port & service discovery

Safe scanning patterns and fast discovery.

nmap / masscan / netcat
# nmap (version + default scripts, full ports)
nmap -sC -sV -p- -T4 --min-rate 1000 -oA nmap-full 192.0.2.0/24

# host discovery
nmap -sn 192.0.2.0/24

# masscan (VERY noisy)
masscan 192.0.2.0/24 -p1-65535 --rate=1000 -oG masscan.gnmap

# service banner grab with netcat
nc -vz 192.0.2.10 80
echo | nc 192.0.2.10 25

4) Web discovery & fuzzing

Directory discovery, vhost fuzzing and parameter bruteforce.

ffuf / gobuster / curl
# ffuf
ffuf -u https://example.com/FUZZ -w /usr/share/wordlists/dirbuster/directory-list-2.3-medium.txt -mc 200,301,302 -o ffuf.json

# gobuster (dir + vhost)
gobuster dir -u https://example.com/ -w /usr/share/wordlists/dirb/common.txt -t 50 -o gobuster.txt
gobuster vhost -u https://example.com -w vhosts.txt -t 30

# bashed URL bruteforce (slow)
while read p; do
  curl -s -o /dev/null -w "%{http_code} %{url_effective}\n" https://example.com/$p
done < wordlist.txt

5) Web application testing & exploitation

HTTP checks, nikto, sqlmap, XSS tooling, and parameter discovery.

curl / nikto / sqlmap / xss
# curl quick checks
curl -I -L https://example.com
curl -s -D - https://example.com/login -o /dev/null

# nikto (webserver checks)
nikto -h https://example.com -output nikto.txt

# sqlmap (ONLY in-scope & lab)
sqlmap -u "https://example.com/vuln.php?id=1" --batch --dbs

# XSS tools
dalfox url "https://example.com/page?param=FUZZ"
xsstrike -u "https://example.com/page?param=1"

# parameter discovery
arjun -u "https://example.com/page" -o arjun.txt

6) Vulnerability scanning & SCA

Fast templates, SCA, SBOM generation and vulnerability lookup.

nuclei / syft / grype / snyk
# nuclei (fast templates)
nuclei -l hosts.txt -t cves/ -o nuclei_results.txt

# syft: generate SBOM
syft packages dir:./project -o json > sbom.json

# grype: scan SBOM or dir
grype dir:./project -o json -o grype.json
grype sbom:sbom.json

# snyk (requires account & token)
snyk test --file=package.json
snyk test --docker myimage:tag

# npm / pip audits
npm audit --json > npm-audit.json
pip-audit -r requirements.txt --format json > pip-audit.json

# Query NVD
curl -s "https://services.nvd.nist.gov/rest/json/cve/1.0/CVE-2021-44228" | jq .

7) Passwords, credentials & AD tooling

Online & offline password tools, impacket and BloodHound.

hydra / john / hashcat / impacket
# hydra (online bruteforce) - AUTHORIZED ONLY
hydra -L users.txt -P passwords.txt -t 4 ssh://192.0.2.10 -o hydra-ssh.txt

# john / hashcat (offline)
john --wordlist=/usr/share/wordlists/rockyou.txt hashfile.txt
hashcat -m 1000 -a 0 ntlm_hashes.txt /usr/share/wordlists/rockyou.txt

# impacket scripts
python3 /opt/impacket/examples/smbclient.py -username user -password pass 192.0.2.15
python3 /opt/impacket/examples/secretsdump.py domain/USER:PASS -outputfile secrets

# BloodHound collection (with rights)
# Invoke-BloodHound -CollectionMethod All -ZipFileName bloodhound.zip

9) Network capture / SSL / TLS checks

pcap capture, tshark analysis and quick TLS checks.

tcpdump / tshark / openssl
# tcpdump (capture 1000 packets)
tcpdump -i eth0 -w capture.pcap -c 1000

# tshark read
tshark -r capture.pcap -Y "http" -T fields -e http.host -e http.request.uri

# quick cert info
openssl s_client -connect example.com:443 -servername example.com /dev/null | openssl x509 -noout -text

# sslyze / testssl.sh
sslyze --regular example.com:443
./testssl.sh example.com

10) Vulnerability Research & PoC Triage

Search public PoCs, triage, isolate and verify safely.

searchsploit / gh / gitleaks / docker
# searchsploit
searchsploit "CVE-2021-44228"
searchsploit --update
searchsploit -m 49993  # copy exploit by EDB ID

# clone exploit-db repo
git clone https://github.com/offensive-security/exploitdb.git
grep -R "CVE-2021-44228" exploitdb -n

# fetch PoC read-only (inspect before running)
curl -sL "https://raw.githubusercontent.com/user/repo/branch/poc.py" -o /tmp/poc.py
sed -n '1,200p' /tmp/poc.py

# run PoC inside an isolated container (no network)
docker run --rm -v /tmp/poc.py:/poc/poc.py --network none -it python:3.10 bash -c "python /poc/poc.py"

11) Binary reversing & exploit development

Static inspection, decompilers and ROP gadget hunting.

readelf / r2 / gdb / ropgadget
# ELF inspection
readelf -h binary
objdump -d binary | less
strings binary

# radare2
r2 -AA binary

# rop gadgets
ropgadget --binary binary --only "pop|ret" > gadgets.txt

# gdb + pwndbg
gdb -q ./binary

12) Fuzzing

Classic fuzzers and HTTP/API fuzzers.

afl / boofuzz / wfuzz
# afl (instrumented binary)
afl-fuzz -i in_dir -o out_dir -- ./target @@

# boofuzz (python)
python3 boofuzz_example.py

# wfuzz for HTTP
wfuzz -c -z file,/usr/share/wordlists/parameters.txt --hc 404 "https://example.com/vuln?FUZZ=1"

13) Mobile & Firmware analysis

APK decompilation, Frida dynamic hooks, firmware extraction.

apktool / jadx / frida / binwalk
# Android decompile / inspect
apktool d app.apk -o app_src
jadx -d out app.apk

# frida dynamic hooking
frida -U -f com.example.app -l script.js --no-pause

# firmware extraction
binwalk -e firmware.bin
strings firmware.bin | grep -i password

14) Safe PoC testing — containers & labs

Isolated containers, snapshots and no-network execution.

docker / vm snapshots
# isolated container (no external network)
docker run --rm --network none -v $(pwd):/work -it ubuntu:22.04 bash

# spin up target service in container
docker run --rm -p 8080:80 vuln-app:latest

# take VM snapshot before PoC; rollback after testing

15) Helpful flags, output tips & logging

Save JSON, reduce noise, capture timestamps and tool versions.

nmap / nuclei / ffuf
# nmap output formats
nmap -sC -sV -oA nmap-output target

# tee logs with timestamps & tool version
(tool --version 2>&1 ; date; tool [args]) |& tee logfile.txt

# reduce scan noise
nmap -T2 --max-retries 2 --host-timeout 30m target
masscan --rate 100

# save JSON where possible
nuclei -o nuclei.json -oJ nuclei.json
ffuf -o ffuf.json -of json
Quick safety & research tips
  • Validate scanner findings manually — many false positives.
  • Isolate PoC execution (containers / VM snapshots).
  • Document CVE IDs, vendor advisories, affected versions, and PoC references.
  • Never weaponize vulnerabilities or test without explicit written permission.
